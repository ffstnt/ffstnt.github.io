anaconda学习笔记二 解决Mac终端jupyter command not found问题

发表于 2019-05-13

因为Mac自带python是2.x版本，我用pip安装了python3.x，用jupyter notebook发现无法使用import pandas, matplotlib等等包，Google了很多，最后我的解决方式如下：

解决jupyter command not found:

  • 安装python3:pip3 install jupyter
  • 在弹出的常常一串字的最后一段出现了我们juypter的安装位置Requirement already satisfied: parso>=0.3.0 in /Library/Frameworks/Python.framework/Versions/3.5/lib/python3.5/site-packages (from jedi>=0.10->ipython>=5.0.0->ipykernel->jupyter) (0.3.1)
  • 在终端输入jupyter notebook提示command not found
  • 接着输入open .bash_profile
  • 在弹出的文本编辑框中输入export PATH=/Library/Frameworks/Python.framework/Versions/3.5/bin/:$PATH输完后按command + s保存。注意：这里的路径需要和自己安装jupyter的位置一致，每个人安装的python版本不一致，所以需要注意查看jupyter安装位置，别输错了。
  • 接着在终端输入source .bash_profile按回车
  • 输入jupyter notebook弹出jupyter notebook网页，成功打开。

解决jupyter notebook无法import pandas等包

  • 在完成上述操作后，在notebook中输入import pandas as pd
    names1880 = pd.read_csv('name/yob1880.txt', names = ['name', 'sex', 'births'])提示files not found，终于不是modules not found了。。
  • 接着试着输入学习笔记一中基础点线图代码，没有报错。之前学习笔记一基础点线图代码我是在jupyter终端中输入的，这个demo可以完美运行，但是在notebook中提示modules not found。
  • 我google了原因，可能是notebook的kernel与终端的kernel不一致，终端能运行，说明终端的kernel没问题，问题出在notebook的kernel上。然后我分别在notebook和jupyter 终端中中输入import sys
    sys.executable两个运行结果不一致。
